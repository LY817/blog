基于kubeadmin搭建k8s实验环境
> 傻瓜式安装 关键是使用国内镜像翻墙

## 创建vmware实例

- 192.168.3.200 master
- 192.168.3.201 worker node
- 192.168.3.202 worker node

## 设置

### 关闭防火墙和swap

```sh
systemctl stop firewalld &&
systemctl disable firewalld &&
sed -i 's/enforcing/disabled/' /etc/selinux/config &&
setenforce 0 &&
swapoff -a &&
sed -ri 's/.*swap.*/#&/' /etc/fstab
```

### 修改host

```sh
hostnamectl set-hostname master[node1/node2]
```

`vim /etc/hosts` 

```
192.168.3.200 master
192.168.3.201 node1
192.168.3.202 node2
```

### 将桥接的IPv4流量传递到iptables的链

```sh
cat > /etc/sysctl.d/k8s.conf << EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
EOF
sysctl --system
```

## 安装

### 修改yum源设置

先安装wget

```sh
yum install -y wget
```

#### 基本源

```sh
wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
```

#### docker

```sh
wget https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo -O /etc/yum.repos.d/docker-ce.repo
```

#### k8s

```sh
cat > /etc/yum.repos.d/kubernetes.repo << EOF
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=0
repo_gpgcheck=0
gpgkey=https://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg https://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
EOF
```

#### 生效配置

```sh
yum clean all
yum makecache
```

### 安装docker

```sh
yum -y install docker-ce-18.06.1.ce-3.el7
systemctl enable docker && systemctl start docker
```

### 安装k8s组件

需要使用aliyun的yum源

```sh
yum install -y kubelet-1.17.0 kubeadm-1.17.0 kubectl-1.17.0
systemctl enable kubelet
```

## 初始化k8s master

```sh
kubeadm init \
--apiserver-advertise-address=192.168.3.200 \
--image-repository registry.aliyuncs.com/google_containers \
--kubernetes-version v1.17.0 \
--service-cidr=10.1.0.0/16 \
--pod-network-cidr=10.244.0.0/16
```

> `echo 1 > /proc/sys/net/ipv4/ip_forward`

- 初始化过程中会拉取并启动docker进行

- 启动成功后会在日志打印相关信息

  - 如何使用kubectl

  ```
  To start using your cluster, you need to run the following as a regular user:
  
    mkdir -p $HOME/.kube
    sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
    sudo chown $(id -u):$(id -g) $HOME/.kube/config
  ```

  - 如何启动一个pod

  ```
  You should now deploy a pod network to the cluster.
  Run "kubectl apply -f [podnetwork].yaml" with one of the options listed at:
    https://kubernetes.io/docs/concepts/cluster-administration/addons/
  ```

  - 如何添加一个新worker节点到这个admin集群 后面命令行中的token需要保存

  ```
  Then you can join any number of worker nodes by running the following on each as root:
  
  kubeadm join 192.168.3.200:6443 --token ty17u0.qulixi9my23ly1iq \
      --discovery-token-ca-cert-hash sha256:a89ede9243d3b57ca571c6cf46134411e7552924d8932232edd8c75c5d85ad38
  ```

## 接入node节点到集群

### 拉取flannel

node节点上的docker 拉取`quay-mirror.qiniu.com/coreos/flannel:v0.12.0-amd64`

### 执行加入命令

```sh
kubeadm join 192.168.3.200:6443 --token ty17u0.qulixi9my23ly1iq \
    --discovery-token-ca-cert-hash sha256:a89ede9243d3b57ca571c6cf46134411e7552924d8932232edd8c75c5d85ad38
```

## 安装控制台

https://kuboard.cn/install/install-dashboard.html

### 获取token

在master节点执行

```sh
echo $(kubectl -n kube-system get secret $(kubectl -n kube-system get secret | grep kuboard-user | awk '{print $1}') -o go-template='{{.data.token}}' | base64 -d)
```
